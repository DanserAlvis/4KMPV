# SHIRO LABS — guía de identidad

Referencia original: sitio web de Shiro Labs.

## Núcleo visual

- Blanco puro: `#ffffff` / paneles de control y superficies de operación.
- Negro: `#050505` / estructura, tipografía principal y contraste.
- Vacío: `#0a0b0e` / fondos técnicos y áreas de contenido.
- Cyan técnico: `#35c9d8` / progreso, foco y pequeños estados interactivos.
- Rojo `#e60000` y naranja / únicamente advertencias, errores o estados excepcionales.
- Gris secundario: `#777777` / metadatos inactivos.

## Lenguaje formal

- La composición debe ser mayoritariamente blanca, negra y gris.
- Cyan, rojo u naranja son acentos pequeños; nunca deben dominar texto, iconos o paneles.
- Geometría industrial, modular y casi rectangular; radios de 2–3 px.
- Bordes finos, divisiones duras y jerarquía clara.
- Tipografía pesada y mayúscula para títulos y acciones.
- Monoespaciada para índices, estados, diagnósticos y metadatos.
- Animaciones breves, funcionales y contenidas.
- Fondos oscuros con retícula técnica apenas visible cuando el medio lo permita.
- Controles agrupados como un bloque físico; estados activos mediante contraste o un pequeño LED/acento.

## Voz del producto

Usar etiquetas concisas de sistema: `PROTOCOLO`, `SYSTEM`, `CORE`, `LINK`,
`ESTADO`, `MÓDULO`, `NEURAL UPSCALE`. Evitar grandes superficies saturadas,
gradientes vistosos, exceso de brillo, esquinas muy redondeadas y decoración gratuita.

Esta guía debe utilizarse para futuras piezas de Shiro Labs dentro de este workspace.
